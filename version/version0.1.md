# Version 0.1 功能规划

## 当前定位

本项目定位为一个基于 LangGraph 多 Agent 工作流的智能旅行助手。核心目标不是只生成一份静态行程，而是逐步演进成可调整、可解释、可互动的旅行规划工作台。

## 已完成能力

### 1. 多 Agent 行程生成

- 景点搜索 Agent
- 天气查询 Agent
- 酒店推荐 Agent
- 行程规划 Agent
- 景点、天气、酒店三路并行收集，汇总后再生成最终行程

### 2. 旅行风格与预算偏好

首页支持选择旅行风格：

- 舒适均衡
- 特种兵模式
- 慢旅行
- 亲子
- 情侣
- 老人友好
- 美食优先
- 拍照打卡

首页支持选择预算档位：

- 经济
- 舒适
- 豪华

### 3. 行程增强信息

后端会在行程生成后自动补齐：

- 天气影响提醒
- 出行打包清单
- 每日路线串联
- 每日路线摘要
- 每日行程评分

每日评分包含：

- 路线紧凑度
- 游玩强度
- 预算友好度
- 天气适配度
- 综合评分

### 4. 结果页交互调整

结果页支持：

- 刷新增强信息
- 某天行程调整为“轻松一点”
- 某天行程调整为“下雨备选”
- 某天行程调整为“路线更顺”
- 一键替换单个景点
- 编辑模式下手动调整景点顺序、删除景点、修改描述

### 5. 展示与导出

结果页支持：

- 行程概览
- 预算明细
- 景点地图
- 每日行程卡片
- 天气信息
- 智能提醒卡片
- 出行清单
- 每日路线串联
- 导出图片
- 导出 PDF

## 下一阶段候选功能

### 1. 行程版本对比

每次用户执行“轻松一点”“路线更顺”“换一个景点”等操作时，自动保存一个版本。

目标能力：

- 查看历史版本
- 回退到任意版本
- 对比两个版本的差异
- 标记当前使用版本

示例：

- 原始计划
- 雨天优化版
- 低预算优化版
- 轻松节奏版

### 2. 旅行偏好记忆

为用户保存长期偏好，下次生成行程时自动带入。

可记忆内容：

- 不喜欢爬山
- 喜欢博物馆
- 预算偏经济
- 每天最多 2 个景点
- 不吃辣
- 需要老人友好路线
- 更喜欢公共交通

### 3. 每日时间轴视图

把每日行程转换为更直观的时间线。

示例：

```text
09:00 早餐
10:00 故宫
12:30 午餐
14:00 景山公园
18:00 晚餐
20:00 返回酒店
```

目标能力：

- 展示每日时间安排
- 标记交通时间
- 标记餐饮时间
- 标记游玩时长
- 为拖拽编辑打基础

### 4. 拖拽式行程编辑

用户可以在前端拖拽景点顺序，拖完后自动重新计算：

- 每日路线
- 路线评分
- 游玩强度
- 天气适配提示

### 5. 行程冲突检测

自动检测行程中的不合理点。

检测项：

- 景点距离太远
- 当天景点太多
- 游玩强度过高
- 午餐或晚餐时间不合理
- 酒店离核心景点太远
- 雨天室外景点过多
- 预算超出当前档位

目标能力：

- 展示冲突原因
- 给出一键修复按钮

### 6. 多方案生成

一次生成多套不同旅行方案，让用户选择。

候选方案：

- 经典必玩版
- 轻松美食版
- 小众深度版
- 低预算版
- 拍照打卡版

目标能力：

- 多方案并排展示
- 每套方案给出评分
- 用户选择一套作为主计划

### 7. 餐厅真实推荐

当前餐饮更多是文本建议。下一步可以基于高德 POI 搜索真实餐厅。

目标能力：

- 根据景点附近推荐早餐、午餐、晚餐
- 展示餐厅地址、评分、距离、预算
- 支持一键替换餐厅
- 支持按口味偏好筛选

### 8. 开放时间与门票提醒

为景点补充更实用的旅行信息。

目标能力：

- 开放时间
- 是否需要预约
- 闭馆风险
- 建议购票时间
- 门票价格
- 节假日拥挤提醒

### 9. 住宿位置智能选择

不仅推荐酒店，还分析应该住在哪个区域。

目标能力：

- 根据每天景点分布推荐住宿商圈
- 分析酒店位置是否合理
- 给出换酒店建议

示例：

```text
建议住在前门/王府井附近，因为第 1-2 天景点集中在核心城区。
```

### 10. 离线旅行包

生成完整可携带的旅行资料。

包含：

- PDF 行程
- 每日路线
- 天气提醒
- 打包清单
- 预算清单
- 酒店地址
- 景点地址
- 紧急联系人/常用电话

## 推荐优先级

### P0

- 每日时间轴视图
- 行程冲突检测
- 餐厅真实推荐

### P1

- 行程版本对比
- 多方案生成
- 拖拽式行程编辑

### P2

- 旅行偏好记忆
- 开放时间与门票提醒
- 住宿位置智能选择
- 离线旅行包

## Version 0.1 目标

Version 0.1 的目标是完成从“静态行程生成器”到“可交互旅行助手”的第一阶段升级。

核心验收标准：

- 用户能生成一份完整行程
- 用户能基于天气、强度、路线对行程进行调整
- 用户能看懂为什么某天行程好或不好
- 用户能获得路线、清单、预算、天气等实用辅助信息
- 用户能导出并保存最终行程
